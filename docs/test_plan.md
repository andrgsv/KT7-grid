# Тестовый план

## 1. Объект тестирования

Веб-сайт Wikipedia: https://www.wikipedia.org/

## 2. Цель тестирования

Проверить возможность удаленного запуска функциональных веб-тестов через Selenium Grid и убедиться, что основные функции сайта работают в разных браузерах.

## 3. Задачи тестирования

1. Настроить Selenium Grid с Hub и несколькими Node.
2. Подключить тесты к Grid через Remote WebDriver.
3. Реализовать тестовые сценарии на Python + Selenium + Pytest.
4. Запустить тесты в Chrome и Firefox.
5. Выполнить параллельный запуск через pytest-xdist.
6. Зафиксировать результаты и возможные проблемы.

## 4. Объем тестирования

Проверяются:

- открытие главной страницы;
- отображение строки поиска;
- поиск статьи;
- отображение языковой навигации;
- открытие английской версии Wikipedia;
- удаленный запуск через Grid;
- параллельное выполнение тестов.

Не проверяются:

- регистрация и авторизация;
- редактирование статей;
- административные функции;
- платежи и персональные данные.

## 5. Типы тестирования

| Тип тестирования | Применяется | Описание |
|---|---|---|
| Функциональное | Да | Проверка функций сайта |
| Кросс-браузерное | Да | Проверка Chrome и Firefox |
| Регрессионное | Да | Повторный запуск сценариев |
| Параллельное | Да | Запуск через pytest-xdist |
| Нагрузочное | Нет | Не входит в задачу |

## 6. Критерии начала тестирования

- Docker установлен и запущен.
- Selenium Grid доступен по адресу http://localhost:4444/ui.
- Узлы Chrome и Firefox зарегистрированы в Grid.
- Установлены зависимости из requirements.txt.
- Тестируемый сайт доступен из браузера.

## 7. Критерии завершения тестирования

- Все запланированные тест-кейсы выполнены.
- Результаты тестирования зафиксированы.
- Обнаруженные проблемы описаны в отчете.
- Сформулированы выводы и рекомендации.

## 8. Риски

| Риск | Влияние | Меры снижения |
|---|---|---|
| Недоступность сайта | Тесты не выполнятся | Повторить запуск позже |
| Node не подключился к Hub | Тесты не стартуют | Проверить Docker и логи контейнеров |
| Различия DOM в браузерах | Возможны падения тестов | Использовать устойчивые локаторы |
| Медленная сеть | Таймауты | Увеличить явные ожидания |
