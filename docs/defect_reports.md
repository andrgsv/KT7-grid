# Отчеты о дефектах и проблемах

В ходе тестирования критические дефекты функциональности сайта Wikipedia не выявлены. Ниже приведены учебные примеры проблем, которые могут возникнуть при настройке удаленного запуска через Selenium Grid.

## DEF-001. Firefox Node не отображается в интерфейсе Grid

**Тип:** проблема тестового окружения  
**Серьезность:** Major  
**Приоритет:** High  
**Статус:** Open

**Описание:**  
После запуска Docker Compose в интерфейсе Selenium Grid отображается Chrome Node, но отсутствует Firefox Node.

**Шаги воспроизведения:**

1. Выполнить команду `docker compose up -d`.
2. Открыть `http://localhost:4444/ui`.
3. Проверить список доступных узлов.

**Ожидаемый результат:**  
В интерфейсе отображаются Chrome Node и Firefox Node.

**Фактический результат:**  
Отображается только Chrome Node.

**Возможная причина:**  
Firefox-контейнер не успел подключиться к Event Bus или завершился с ошибкой.

**Рекомендации:**

- выполнить `docker compose ps`;
- проверить логи `docker compose logs firefox`;
- перезапустить Grid командой `docker compose down && docker compose up -d`.

## DEF-002. Тест поиска падает из-за таймаута загрузки страницы

**Тип:** нестабильность автотеста / внешняя зависимость  
**Серьезность:** Minor  
**Приоритет:** Medium  
**Статус:** Open

**Описание:**  
При медленном интернет-соединении тест поиска может завершиться ошибкой ожидания элемента или страницы.

**Шаги воспроизведения:**

1. Запустить Grid.
2. Выполнить `pytest -n 2 --browser chrome,firefox`.
3. Дождаться выполнения теста поиска.

**Ожидаемый результат:**  
Страница с результатами поиска открывается в пределах таймаута.

**Фактический результат:**  
При нестабильной сети возможен timeout.

**Возможная причина:**  
Зависимость от внешнего сайта и качества сети.

**Рекомендации:**

- увеличить время явных ожиданий;
- повторить тест;
- при необходимости использовать локальный тестовый сайт.
